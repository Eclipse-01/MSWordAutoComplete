## 安装

### 开发环境安装

1. 克隆仓库并安装依赖：
```bash
git clone https://github.com/Eclipse-01/MSWordAutoComplete.git
cd MSWordAutoComplete
npm install
```

2. 启动开发服务器：
```bash
npm start
```

这将自动在 Word 中侧载插件。

### 生产环境部署

1. 构建项目：
```bash
npm run build
```

2. 部署 `dist` 文件夹到您的 Web 服务器

3. 更新 `manifest.xml` 中的 URL 为您的服务器地址

4. 在 Word 中手动侧载 manifest.xml 文件

## 首次配置

### 1. 获取 API 密钥

#### 选项 A：使用 OpenAI

1. 访问 [OpenAI 平台](https://platform.openai.com/)
2. 注册或登录账户
3. 导航到 API 密钥页面
4. 创建新的 API 密钥并复制

#### 选项 B：使用 GitHub Copilot

1. 确保您有 GitHub Copilot 订阅
2. 访问 [GitHub Settings](https://github.com/settings/tokens)
3. 创建具有 Copilot 访问权限的个人访问令牌
4. 复制令牌

#### 选项 C：使用自定义 API

如果您有自己的 OpenAI 兼容 API 服务：
1. 准备好 API 端点 URL
2. 准备好 API 密钥

### 2. 在插件中配置

1. 在 Word 中打开插件
2. 点击"设置"选项卡
3. 选择您的 API 提供商
4. 输入 API 密钥或令牌
5. 配置其他参数：
   - **模型**：选择要使用的 AI 模型
   - **温度值**：0.0-2.0，控制输出的随机性
   - **最大 Token 数**：控制生成文本的长度
6. 点击"保存设置"

## 功能详解

### 🤖 自动补全

自动补全功能会在您键入时自动提供 AI 生成的文本建议。

**使用方法：**

1. 切换到"自动补全"选项卡
2. 启用"启用自动补全"开关
3. 选择触发模式：
   - **自动**：在您键入超过 10 个字符后自动触发
   - **手动**：点击"获取建议"按钮手动触发
4. 在文档中键入文本
5. 查看建议，点击"应用建议"将其插入文档

**最佳实践：**
- 至少输入一段完整的句子或段落开头
- 提供足够的上下文以获得更好的建议
- 手动模式适合需要精确控制的场景

### 💬 内联聊天

内联聊天允许您与 AI 讨论选中的文本，获取改进建议。

**使用方法：**

1. 在 Word 文档中选中一段文本
2. 切换到"内联聊天"选项卡
3. 插件会自动加载选中的文本
4. 在输入框中输入您的请求，例如：
   - "改进这段文字的表达"
   - "使这段话更正式"
   - "总结这段内容"
   - "翻译成英文"
5. 点击"发送"或按 Enter
6. 查看 AI 响应，点击"应用到文档"将其插入

**提示：**
- 可以多次来回对话以精炼内容
- 点击"刷新选择"更新选中的文本

### 🗨️ 侧边聊天

侧边聊天提供完整的对话界面，适合长对话和复杂任务。

**使用方法：**

1. 切换到"侧边聊天"选项卡
2. 在输入框中输入您的问题或请求
3. AI 会考虑文档上下文（如果有选中文本）
4. 查看响应并继续对话
5. 点击"插入到文档"将响应内容添加到文档

**应用场景：**
- 头脑风暴和创意生成
- 文档大纲规划
- 研究和资料整理
- 写作指导和建议

**功能：**
- 支持多轮对话
- 保持对话历史
- 点击"清空"开始新对话

### ⚙️ 设置

配置 API 提供商、模型参数等。

**可配置项：**

- **API 提供商**：GitHub Copilot、OpenAI、自定义
- **API 密钥/令牌**：用于认证
- **模型**：选择 AI 模型（GPT-3.5、GPT-4 等）
- **温度值**：
  - 0.0-0.3：更专注、确定性的输出
  - 0.4-0.7：平衡的输出（推荐）
  - 0.8-2.0：更有创造性、多样性的输出
- **最大 Token 数**：
  - 较小值（100-300）：短回复
  - 中等值（300-800）：段落级回复（推荐）
  - 较大值（800+）：长文本生成

## 常见问题

### Q: 为什么我收到"无法从 AI 服务获取响应"错误？

**A:** 可能的原因：
1. API 密钥无效或过期
2. API 配额已用完
3. 网络连接问题
4. API 服务暂时不可用

**解决方法：**
- 检查设置中的 API 密钥
- 验证 API 账户状态和配额
- 检查网络连接
- 稍后重试

### Q: 建议的内容不够准确或相关？

**A:** 尝试：
1. 提供更多上下文
2. 使用更具体的提示
3. 调整温度值（降低以获得更专注的输出）
4. 尝试不同的模型

### Q: 插件加载很慢？

**A:** 这可能是由于：
1. 首次加载时需要下载依赖
2. API 响应时间
3. 网络速度

**优化建议：**
- 使用较小的 max tokens 值
- 确保良好的网络连接
- 考虑使用更快的 API 端点

### Q: 如何保护我的 API 密钥？

**A:** 
- API 密钥存储在浏览器的本地存储中
- 不要与他人分享您的 API 密钥
- 定期轮换 API 密钥
- 在 API 提供商处设置使用限制

### Q: 这个插件与 Microsoft 有官方关系吗？

**A:** 
不，这是一个第三方开发的插件，与 Microsoft Corporation 无任何官方关联。

## 提示与技巧

### 💡 获得更好的自动补全

1. **提供清晰的上下文**：确保前面的文本清楚地表达了您的意图
2. **使用一致的风格**：AI 会学习您的写作风格
3. **适时停顿**：在完成一个想法后停止，让 AI 继续
4. **迭代改进**：如果第一次建议不理想，稍作调整再试

### 💡 高效使用聊天功能

1. **明确指令**：
   - ❌ "改进这个"
   - ✅ "使这段话更正式，适合商务邮件"

2. **分步骤**：复杂任务分解为多个小步骤

3. **提供示例**：告诉 AI 您想要的输出风格

4. **利用对话历史**：在侧边聊天中，AI 会记住之前的对话

### 💡 优化性能

1. **调整 Token 限制**：根据需求使用合适的值
2. **使用手动触发**：在自动补全中使用手动模式以节省 API 调用
3. **缓存常用响应**：对于重复的任务，保存 AI 的响应

### 💡 创意应用场景

1. **内容创作**：
   - 博客文章大纲
   - 社交媒体文案
   - 营销材料

2. **学术写作**：
   - 文献综述辅助

   - 摘要生成

3. **商务文档**：
   - 报告撰写
   - 提案准备

4. **创意写作**：
   - 故事情节发展
   - 角色对话
   - 场景描述
